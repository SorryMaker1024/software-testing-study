# 软件测试学习计划（一个月高强度）

## 基本情况
- 2025年计算机专业学院本科毕业
- 在宁波，目标一个月后找到测试相关工作
- 每天 8-10 小时，每周 6 天，休息 1 天

---

## 第一周：测试地基

### 第1-2天
- B站搜「软件测试基础」，找黑马程序员或尚硅谷的教程
- 学：测试流程、黑盒/白盒/灰盒、功能/性能/兼容/安全测试、单元/集成/系统/验收测试
- 目标：面试时能说清楚"从需求到上线，测试在哪个阶段做什么"

### 第3-4天
- B站搜「测试用例设计方法」，推荐柠檬班
- 学：等价类划分、边界值分析、因果图、场景法、错误推测法
- 动手：给一个登录页写出 20 条测试用例（正常登录、密码错误、空值、SQL注入、验证码异常等）

### 第5-6天
- SQL强化：牛客网刷 SQL 入门+进阶共 50 题
- 学：SELECT/WHERE/JOIN/GROUP BY/HAVING/子查询/聚合函数

### 第7天
- Bug生命周期 + 禅道/Jira 实操
- 本周复习

---

## 第二周：自动化测试（核心）

### 第8-9天
- Python 速成：变量、if/for/while、list/dict/tuple、函数、类

### 第10-11天
- Selenium WebDriver：8种元素定位、等待机制、窗口切换
- 动手：自动化打开淘宝搜索"机械键盘"，翻到第3页，截图

### 第12-13天
- pytest 框架：用例编写、断言、fixture、参数化、Allure 报告

### 第14天
- 整合项目：Selenium + pytest 写 5 条以上自动化用例，代码推到 GitHub

---

## 第三周：接口测试

### 第15-16天
- HTTP 协议基础 + Postman 使用

### 第17-18天
- Python Requests 库 + 接口自动化
- 动手：对公开 API 写 10 条自动化接口用例

### 第19-20天
- JMeter 入门（了解即可）

### 第21天
- 复习 + 代码整理 + GitHub 提交

---

## 第四周：简历 + 面试 + 海投

### 第22-23天
- 构建简历项目，整理代码，写 README，推到 GitHub

### 第24-25天
- 刷面试题：测试理论50题 + SQL30题 + Python基础30题

### 第26-27天
- 模拟面试：自我介绍、项目介绍、最难的Bug、测试流程

### 第28-30天
- BOSS直聘每天沟通 50-80 个岗位
- 面试 + 复盘

---

## 备选方案：实施工程师
如果第四周一个面试都没有，立刻切换：
- 宁波做 ERP/WMS 的公司（聚水潭、旺店通、万里牛等）常年招实施
- 一周准备周期，突出"计算机专业 + 熟悉数据库 + 懂电商业务流程"

## 简历改造方向
把原有两个项目从测试角度重写：

### 项目一：Vue + SpringBoot 管理系统
改为：XX管理系统 — 功能测试与自动化测试实践
- 设计测试用例 80+ 条
- Selenium + Python 自动化脚本 15 条
- Postman 接口测试
- 禅道缺陷管理，发现 Bug 20+ 个

### 项目二：OCR 识别系统
改为：OCR识别系统 — 接口测试与数据验证
- 不同图片格式、分辨率场景测试方案
- Python Requests 自动化接口用例
- 准确率统计分析，输出测试报告
