# 接口测试速成笔记

> 接口测试 = 不通过浏览器，直接用代码向服务器发 HTTP 请求，验证返回结果。
> 面试必问 + 工作每天都在用。

---

## 一、HTTP 是什么？

浏览器访问网页的过程：

```
浏览器  →  发 HTTP 请求 →  服务器
浏览器  ←  返回 HTML 页面 ←  服务器
```

接口测试就是**跳过浏览器**，直接发请求、看返回数据：

```
代码  →  发 HTTP 请求 →  服务器
代码  ←  返回 JSON 数据 ←  服务器
```

---

## 二、HTTP 请求的结构（4 部分）

```
POST https://api.example.com/login  HTTP/1.1        ← 1. 方法 + URL
Content-Type: application/json                        ← 2. Headers（请求头）
Authorization: Bearer xxxxxxxx

{
  "username": "admin",                                ← 3. Body（请求体，GET 没有）
  "password": "123456"
}
```

### 1. 请求方法（Method）

| 方法 | 作用 | 类比 |
|------|------|------|
| GET | 获取数据 | 看菜单 |
| POST | 新增数据 | 下单 |
| PUT | 修改数据（全量） | 换菜 |
| PATCH | 修改数据（部分） | 加辣 |
| DELETE | 删除数据 | 退菜 |

工作中 90% 只用 **GET** 和 **POST**。

### 2. URL

```
https://api.example.com/v1/users?page=1&size=10
  ↑        ↑            ↑    ↑        ↑
  协议     域名         路径   参数  参数值

? 后面的叫 Query String（查询参数），格式：key=value&key=value
```

### 3. Headers（请求头）

告诉服务器额外信息：

```json
Content-Type: application/json    ← 我发的是 JSON 格式
Authorization: Bearer xxx         ← 我的身份令牌
User-Agent: Mozilla/5.0           ← 我是什么浏览器
```

### 4. Body（请求体）

GET 没有 Body，POST/PUT 有：

```json
{
  "username": "admin",
  "password": "123456"
}
```

## 三、HTTP 响应的结构

```
HTTP/1.1 200 OK                                       ← 状态码
Content-Type: application/json                        ← 响应头

{
  "code": 0,                                          ← 响应体（Body）
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOi..."
  }
}
```

### 常见状态码

| 状态码 | 含义 | 场景 |
|--------|------|------|
| 200 | OK，成功 | 请求正常 |
| 201 | Created，创建成功 | POST 新增后返回 |
| 301 | 永久重定向 | 网址迁移 |
| 400 | Bad Request，请求参数错误 | 少传了必填字段 |
| 401 | Unauthorized，未认证 | 没登录或 token 过期 |
| 403 | Forbidden，没权限 | 登录了但不让你看 |
| 404 | Not Found，资源不存在 | URL 写错了 |
| 500 | 服务器内部错误 | 后端崩了 |

口诀：2xx = 成功，4xx = 你的问题，5xx = 服务器的问题

## 四、Postman 是什么

GUI 工具，用来手动发 HTTP 请求、看返回结果。

**安装**：https://www.postman.com/downloads/

### 核心操作（装好后跟着点一遍）

1. 新建 Collection（用来管理一组接口）
2. 在 Collection 里新建 Request
3. 选方法（GET/POST），填 URL，有需要时填 Headers 和 Body
4. 点 Send
5. 看下面的 Response（状态码 + 返回的 JSON）

### 用 Postman 自带的 Demo 练手

```
GET https://postman-echo.com/get?name=test&age=20
```

这个不需要安装任何东西，直接发 GET 请求，看看返回什么。

## 五、Python Requests（自动化接口测试的核心）

### 安装

```bash
pip install requests
```

### GET 请求

```python
import requests

# 发 GET 请求
res = requests.get("https://postman-echo.com/get", params={"name": "test", "age": 20})

# 看结果
print(res.status_code)  # 200
print(res.json())        # 服务器返回的 JSON
```

`params` 会自动拼到 URL 后面变成 `?name=test&age=20`。

### POST 请求

```python
# 发 POST 请求，body 是 JSON
res = requests.post(
    "https://postman-echo.com/post",
    json={"username": "admin", "password": "123456"}
)

print(res.status_code)  # 200
print(res.json())        # 服务器原样返回你发的 body
```

### 响应内容怎么读

```python
res = requests.get("https://api.example.com/data")

res.status_code   # 200 — 状态码（int）
res.text          # 原始文本（str）
res.json()        # 解析成 dict/list（常用，前提是返回的是 JSON）
res.headers       # 响应头（dict）
```

### 带 Headers 的请求（认证）

```python
headers = {
    "Authorization": "Bearer eyJhbGciOi...",
    "Content-Type": "application/json"
}
res = requests.get("https://api.example.com/private", headers=headers)
```

## 六、接口测试 = assert + requests

```python
import requests

def test_get_user():
    res = requests.get("https://api.example.com/users/1")

    assert res.status_code == 200, f"状态码应为200，实际为{res.status_code}"
    data = res.json()
    assert "username" in data, "返回数据应包含 username 字段"
    assert isinstance(data["id"], int), "id 应为整数"
```

跟 Selenium+pytest 的思路一模一样：操作 → assert 验证。
