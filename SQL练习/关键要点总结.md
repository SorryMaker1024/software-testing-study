# SQL 关键要点总结

> 学习期间反复问过的、容易踩坑的知识点

---

## 1. LEFT JOIN 里 AND 和 WHERE 的区别

**最核心的区别**：AND 写在 ON 里是"怎么连"，WHERE 是"留哪些行"。

```sql
-- ✅ 正确：条件在 ON 里，配不上的行保留为 NULL
LEFT JOIN table2
    ON t1.id = t2.id
    AND t2.date BETWEEN '2021-08-01' AND '2021-08-30'

-- ❌ 错误：条件在 WHERE 里，NULL 行被干掉，退化成 INNER JOIN
LEFT JOIN table2 ON t1.id = t2.id
WHERE t2.date BETWEEN '2021-08-01' AND '2021-08-30'
```

**口诀**：ON 里的是"我想这样连接"，WHERE 是"连不上就滚"。

**典型场景**：算留存率、算"没做过某事的用户"——条件必须写在 ON 里。

---

## 2. GROUP BY 的 SELECT 规则

SELECT 里的每个列，都要满足**一个**条件：

| 条件 | 例子 |
|------|------|
| 在 GROUP BY 里 | `GROUP BY university` → `SELECT university` ✅ |
| 被聚合函数包着 | `SELECT MIN(gpa)` ✅ |
| 两个都不满足 | `SELECT device_id` ❌ 数据库不知道选哪个 |

**典型错误**：
```sql
-- ❌ device_id 不在 GROUP BY 里，也没聚合
SELECT device_id, university, MIN(gpa)
FROM user_profile
GROUP BY university
```

## 3. 多列子查询

当需要同时匹配多个字段时，（col1, col2） 绑定匹配：

```sql
-- ✅ 学校和最低分同时对应
WHERE (university, gpa) IN (
    SELECT university, MIN(gpa)
    FROM user_profile
    GROUP BY university
)

-- ❌ 拆开会张冠李戴 — 分数只要跟"任意学校"最低分一样就留下
WHERE university IN (...) AND gpa IN (...)
```

## 4. SUM + CASE WHEN 做条件计数

```sql
-- 统计满足条件的行数
SUM(CASE WHEN result = 'right' THEN 1 ELSE 0 END) AS right_cnt

-- 等价于（但 SUM 方式更灵活）
COUNT(CASE WHEN result = 'right' THEN 1 END)
```

CASE WHEN 在 SUM 里就像拿一叠纸，满足条件记1、不满足记0，最后SUM加总。

---

## 5. ON 和 WHERE 的执行顺序

SQL 执行顺序（逻辑上的，不是物理上的）：

```
FROM → ON → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
```

关键：LEFT JOIN 的 ON 在 JOIN 之前发挥作用，WHERE 在 JOIN 之后。所以 WHERE 能把 LEFT JOIN 的 NULL 行杀掉。

## 6. SUBSTRING_INDEX 字符串解析

```sql
SUBSTRING_INDEX(str, delimiter, N)
```

- N 正数：从左往右取第 N 个分隔符之前
- N 负数：从右往左取第 N 个分隔符之后

```sql
-- 'www.baidu.com/username' → 'username'
SUBSTRING_INDEX(blog_url, '/', -1)

-- 嵌套提取中间字段：'123,male,21,Peking' → '21'（取倒数第二段）
SUBSTRING_INDEX(SUBSTRING_INDEX(profile, ',', -2), ',', 1)
```

## 7. LIKE vs REGEXP

| | LIKE | REGEXP |
|------|------|--------|
| 能模糊匹配 | ✅ | ✅ |
| 能写正则 | ❌ | ✅ |
| 多条件OR | `LIKE '%A%' OR LIKE '%B%'` | `REGEXP 'A|B'` |
| 格式校验 | ❌ 做不到 | ✅ `REGEXP '^1[3-9]\d{9}$'` |

---

## 8. UNION vs UNION ALL

| | 是否去重 | 性能 |
|------|---------|------|
| UNION | 去重 | 慢（要排序比较） |
| UNION ALL | 不去重 | 快 |

除非明确要去重，否则用 UNION ALL。

---

## 9. 聚合函数 + NULL

- `COUNT(*)` — 统计所有行，包括 NULL
- `COUNT(column)` — 只统计非 NULL 值
- `SUM` / `AVG` / `MIN` / `MAX` — 自动忽略 NULL

```sql
-- COUNT(q2.device_id) = 2（NULL 不算）
-- COUNT(q1.device_id) = 3
-- 留存率 = 2/3 ✅
```

---

## 10. 窗口函数（SUM OVER）

```sql
-- 累计求和：按日期顺序，每一行 = 从开始到当前的累加和
SUM(profit) OVER (ORDER BY profit_date) AS cumulative_profit

-- 分组累计求和：每个产品各自累计
SUM(profit) OVER (PARTITION BY product_id ORDER BY profit_date)
```

窗口函数不改变行数，只是给每行多算一个值。
