## 快速开始

### 启动被测系统
```bash
cd 简历项目-后台管理系统
node "C:\Users\蒋天峰\AppData\Roaming\npm\node_modules\pnpm\bin\pnpm.cjs" dev
# 访问 http://localhost:3002
# 登录：admin/123456，验证码填 0
```

### 运行 Selenium 自动化
```bash
cd selenium
pip install -r requirements.txt
pytest tests/ -v --html=report.html
```

### 导入 Postman
1. Postman → Import → 选择 `postman/vue3-admin-postman-collection.json`
2. Import → 选择 `postman/vue3-admin-postman-environment.json`
3. 先跑"获取验证码"→"登录"获取 Token，再跑其他接口

## 关键技术实践

| 技术点 | 说明 |
|--------|------|
| **Mock 验证码改造** | 原验证码为装饰性展示，改造为后端实际校验（9×0=? → 填"0"） |
| **multipart → JSON** | 排查 `vite-plugin-mock-dev-server` 的 `co-body` 无法解析 multipart，改为 JSON 通信 |
| **Page Object 模式** | Selenium 用例采用 POM 设计模式，封装 Element Plus 组件操作 |
| **失败自动截图** | conftest.py 的 pytest hook 在用例失败时自动保存截图 |
| **Postman 环境变量** | 通过 `{{token}}` 实现链式调用，登录→CRUD 流程自动化 |
